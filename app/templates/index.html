Endpoints Sandbox

1. Authentication & RBAC

Practice JWT login, HTTP-only cookies, and role-based permissions.

Authentication
POST /api/v1/auth/login
POST /api/v1/auth/logout
GET /api/v1/auth/me
  • admin / password123 (Access: Full Read/Create/Update/Delete)
  • tester / testpass (Access: Read/Create/Update)
  • user / userpass (Access: Read only)

2. Products CRUD (In-Memory)

Standard RESTful operations with pagination, query filtering, and role validations.

CRUD
GET /api/v1/products?skip=0&limit=10&category=...
GET /api/v1/products/{id}
POST /api/v1/products
PUT /api/v1/products/{id}
DELETE /api/v1/products/{id}
POST /api/v1/products/reset
  • Categories: Electronics, Footwear, Kitchenware, Books, Clothing
  • Create & Update require tester or admin. Delete requires admin.
  • Use /reset to reload the original 4 mock products.

3. File Uploads

Test binary file uploads. Validates mime types, sizes, and extracts metadata.

Multipart
POST /api/v1/upload/image
POST /api/v1/upload/text
  • Image Upload: Limits to 5MB. Checks formatting and extracts size details.
  • Text Upload: Decodes and analyzes line, character, and word count.

4. Codec & Token Utilities

Verify encoding/decoding and token payloads.

Utilities
POST /api/v1/codec/base64/encode
POST /api/v1/codec/base64/decode
POST /api/v1/codec/jwt/decode

5. Edge-Case Simulators

Simulate delays (timeouts), error response codes, rate limits, and header reviews.

Simulators
GET /api/v1/simulate/delay/{seconds}
GET /api/v1/simulate/status/{code}
GET /api/v1/simulate/headers
GET /api/v1/simulate/rate-limit
